# SP Vision 25 启动流程

## 项目概述

SP Vision 25是一个完整的RoboMaster竞赛视觉系统，包含自动瞄准、能量机关击打、全向感知等功能模块。本文档将介绍如何启动和测试该系统，特别是自动瞄准功能。

## 环境要求

- Ubuntu 20.04 LTS
- C++ 17 编译器
- CMake 3.16.3+
- OpenCV
- Eigen3
- fmt
- spdlog
- yaml-cpp
- nlohmann_json
- Threads
- OpenVINO (可选，用于神经网络推理)

## 编译步骤

1. **进入项目目录**
   ```bash
   cd ~/sp_vision_25-main
   ```

2. **创建并进入build目录**
   ```bash
   mkdir -p build
   cd build
   ```

3. **配置CMake**
   ```bash
   cmake ..
   ```

4. **编译项目**
   ```bash
   make -j$(nproc)
   ```

## 配置文件说明

### 主要配置文件

- **configs/standard3.yaml**：标准自瞄配置
- **configs/standard4.yaml**：备用自瞄配置
- **configs/demo.yaml**：演示配置
- **configs/sentry.yaml**：哨兵配置
- **configs/uav.yaml**：无人机配置

### 关键配置参数

1. **敌我方颜色**
   ```yaml
   enemy_color: "red"  # 敌方颜色，可选 "red" 或 "blue"
   ```

2. **相机参数**
   ```yaml
   camera_name: "hikrobot"  # 相机名称
   exposure_ms: 2.5          # 曝光时间(ms)
   gain: 16.9               # 增益
   vid_pid: "2bdf:0001"     # 相机VID:PID
   ```

3. **检测参数**
   ```yaml
   use_traditional: true    # 是否使用传统检测方法
   min_confidence: 0.8       # 神经网络检测置信度阈值
   ```

4. **射击参数**
   ```yaml
   auto_fire: true           # 是否自动射击
   first_tolerance: 3        # 近距离射击容差(度)
   second_tolerance: 2       # 远距离射击容差(度)
   ```

## 测试方法

### 自动瞄准测试

1. **准备工作**
   - 确保相机已正确连接
   - 确保配置文件中的相机参数正确
   - 准备一块装甲板作为测试目标

2. **启动程序**
   ```bash
   cd ~/sp_vision_25-main/build
   ./standard configs/standard3.yaml
   ```

3. **测试步骤**
   - 将装甲板放在相机视野范围内
   - 观察终端输出，查看是否成功检测到装甲板
   - 调整装甲板位置，测试不同距离和角度下的检测效果
   - 观察程序是否能稳定跟踪装甲板

### 常用调试技巧

1. **调整相机曝光**
   ```yaml
   exposure_ms: 3.0  # 增加曝光时间，适用于低光照环境
   gain: 20.0        # 增加增益，适用于低光照环境
   ```

2. **调整检测区域**
   ```yaml
   use_roi: true     # 启用ROI
   roi:
     x: 400
     y: 100
     width: 640
     height: 480
   ```

3. **调整检测参数**
   ```yaml
   threshold: 180    # 调整二值化阈值
   min_lightbar_ratio: 2.0  # 调整灯条比例
   ```

4. **切换检测方法**
   ```yaml
   use_traditional: false  # 使用神经网络检测
   ```

## 常见问题及解决方法

### 1. 相机无法打开

**问题**：程序启动后无法打开相机，提示"Camera open failed"

**解决方法**：
- 检查相机是否正确连接
- 检查配置文件中的相机名称和VID:PID是否正确
- 检查相机驱动是否已安装
- 尝试调整相机曝光和增益参数

### 2. 无法检测到装甲板

**问题**：相机能正常打开，但无法检测到装甲板

**解决方法**：
- 检查敌我方颜色设置是否正确
- 调整检测阈值和灯条比例参数
- 确保装甲板在相机视野范围内
- 检查光照条件，调整相机曝光参数
- 尝试切换检测方法（传统方法/神经网络）

### 3. 检测不稳定

**问题**：能检测到装甲板，但检测结果不稳定，经常丢失目标

**解决方法**：
- 调整tracker参数，增加`min_detect_count`，减少`max_temp_lost_count`
- 调整相机曝光，确保图像清晰
- 检查装甲板是否有反光或遮挡
- 尝试调整ROI区域，缩小检测范围

### 4. 编译错误

**问题**：编译过程中出现错误

**解决方法**：
- 确保所有依赖库已正确安装
- 检查CMake版本是否符合要求
- 检查编译器是否支持C++ 17
- 查看详细错误信息，根据错误提示修复问题

### 5. 神经网络推理错误

**问题**：使用神经网络检测时出现错误

**解决方法**：
- 确保OpenVINO已正确安装
- 检查模型文件是否存在于assets目录下
- 检查配置文件中的模型路径是否正确
- 尝试使用CPU设备进行推理

## 功能模块说明

### 自动瞄准 (auto_aim)

- **功能**：识别敌方装甲板并进行自动瞄准
- **主要文件**：`tasks/auto_aim/`目录下的文件
- **可执行程序**：standard, mt_standard, standard_mpc等

### 能量机关 (auto_buff)

- **功能**：识别并击打能量机关（大符）
- **主要文件**：`tasks/auto_buff/`目录下的文件
- **可执行程序**：auto_buff_debug, auto_buff_debug_mpc等

### 全向感知 (omniperception)

- **功能**：提供全向视觉感知能力
- **主要文件**：`tasks/omniperception/`目录下的文件
- **可执行程序**：sentry系列（需ROS环境）
